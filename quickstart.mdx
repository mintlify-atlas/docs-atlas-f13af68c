# Quick start

Get Wren LSP up and running in your editor in just a few minutes.

## Prerequisites

<Steps>
  <Step title="Install Wren LSP">
    Download and install the Wren LSP binary for your platform. See the [installation guide](/installation) for detailed instructions.

    Verify the installation:
    ```bash
    wren-lsp --help
    ```
  </Step>

  <Step title="Choose your editor">
    Wren LSP works with any editor that supports the Language Server Protocol. This guide uses Neovim as an example, but you can use VS Code, Helix, Sublime Text, or any other LSP-compatible editor.

    See the [editor setup guide](/editors/neovim) for configuration instructions for other editors.
  </Step>
</Steps>

## Neovim setup

This example shows how to configure Wren LSP in Neovim with `nvim-lspconfig`.

<Steps>
  <Step title="Register the Wren filetype">
    Add Wren filetype detection to your Neovim configuration:

    ```lua init.lua
    vim.filetype.add({
      extension = {
        wren = "wren",
      },
    })
    ```

    This tells Neovim to recognize `.wren` files.
  </Step>

  <Step title="Configure the LSP client">
    If you're using `nvim-lspconfig` (recommended), add this to your configuration:

    ```lua init.lua
    local lspconfig = require("lspconfig")
    local configs = require("lspconfig.configs")

    if not configs.wren_lsp then
      configs.wren_lsp = {
        default_config = {
          cmd = { "wren-lsp" },
          filetypes = { "wren" },
          root_dir = lspconfig.util.root_pattern("wren-lsp.json", ".git"),
        },
      }
    end

    lspconfig.wren_lsp.setup({})
    ```

    This configuration:
    - Sets the command to launch the LSP server
    - Activates for `.wren` files
    - Detects the project root by looking for `wren-lsp.json` or `.git`
  </Step>

  <Step title="Optional: Configure without lspconfig">
    If you prefer not to use `nvim-lspconfig`, you can configure the LSP client directly:

    ```lua init.lua
    vim.filetype.add({
      extension = {
        wren = "wren",
      },
    })

    vim.api.nvim_create_autocmd("FileType", {
      group = vim.api.nvim_create_augroup("wren", {}),
      pattern = "wren",
      callback = function()
        vim.lsp.start({
          name = "Wren LSP",
          cmd = { "wren-lsp" },
          root_dir = vim.loop.cwd(),
          flags = { exit_timeout = 1000 },
        })
      end,
    })
    ```
  </Step>

  <Step title="Restart Neovim">
    Restart Neovim for the changes to take effect.
  </Step>
</Steps>

## Test the setup

Create a test file to verify that Wren LSP is working:

<Steps>
  <Step title="Create a Wren file">
    Create a new file called `test.wren`:

    ```bash
    touch test.wren
    nvim test.wren
    ```
  </Step>

  <Step title="Add sample code">
    Type the following Wren code:

    ```wren test.wren
    class Calculator {
      static add(a, b) {
        return a + b
      }

      static multiply(a, b) {
        return a * b
      }
    }

    System.print(Calculator.add(5, 3))
    ```
  </Step>

  <Step title="Test LSP features">
    Try these LSP features to verify everything works:

    **Completion**
    - Type `Calculator.` and wait for suggestions
    - You should see `add` and `multiply` methods

    **Go to definition**
    - Place your cursor on `Calculator` in the `System.print` line
    - Press `gd` (or your editor's go-to-definition keybind)
    - Your cursor should jump to the class definition

    **Hover information**
    - Hover over the `add` method call
    - You should see parameter information

    **Diagnostics**
    - Add a syntax error like `class Broken {`
    - You should see an error diagnostic
  </Step>
</Steps>

## Configure module resolution

For projects with custom module resolution, create a `wren-lsp.json` configuration file in your project root:

```json wren-lsp.json
{
  "version": 1,
  "modules": ["./src"],
  "resolvers": [
    {
      "type": "path",
      "roots": ["./src"],
      "delimiter": "/"
    }
  ]
}
```

This configuration:
- Defines where module files are located
- Specifies how import paths are resolved
- Supports multiple resolver types: `path` and `plugin`

<Info>
Learn more about module resolution in the [configuration guide](/configuration/module-resolution).
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="LSP not starting">
    Verify that `wren-lsp` is in your PATH:
    ```bash
    which wren-lsp  # Linux/macOS
    where wren-lsp  # Windows
    ```

    Check your editor's LSP logs for error messages. In Neovim:
    ```vim
    :LspLog
    ```
  </Accordion>

  <Accordion title="No completions or diagnostics">
    Ensure the LSP client has started:
    - In Neovim: `:LspInfo`
    - Verify that `wren_lsp` is attached to the buffer

    Check that your file has the `.wren` extension and is recognized as a Wren file:
    ```vim
    :set filetype?
    ```
  </Accordion>

  <Accordion title="Module imports not resolving">
    Create a `wren-lsp.json` file in your project root with the appropriate resolver configuration. See the [configuration guide](/configuration/overview) for details.
  </Accordion>

  <Accordion title="macOS blocks the binary">
    macOS Gatekeeper may prevent the binary from running. Allow it in:
    **System Settings** → **Privacy & Security** → **Allow Anyway**

    Or remove the quarantine attribute:
    ```bash
    xattr -d com.apple.quarantine /usr/local/bin/wren-lsp
    ```
  </Accordion>
</AccordionGroup>
